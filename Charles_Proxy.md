Charles_Proxy <br>
-------------------------------------------
용도: 데이터 조작을 통한 UI/UX 엣지 케이스 검증 / 네트워크 환경 시뮬레이션
<br>
<br>

1. Rewrite 기능: 실시간 데이터 조작 및 가상 시나리오 테스트<br>
디바이스 화면 내 특정 텍스트 변형 및 API 응답값(Response Body)을 가로채서 원하는 값으로 변조하는 기능<br>
주요 용도:<br>
- 텍스트 변형: 앱/웹 화면에 노출되는 UI 문구(예: 상품명, 가격, 안내 메시지)를 강제로 변경하여 레이아웃 깨짐이나 폰트 적용 상태 확인.<br>
- 데이터 조작: 서버 데이터를 수정하지 않고도, 특정 조건을 만족하는 데이터(예: 애플 주가 $999.99, 보유 자산 100억 등)를 클라이언트에 내려주어 특정 UI가 잘 노출되는지 테스트.<br>
- 에러 케이스 강제 발생: 정상 응답을 404 Not Found나 500 Error로 바꿔서 앱의 에러 핸들링 로직이 잘 작동하는지 검증.<br>

<br>
<br>

2. 네트워크 쓰로틀링(Network Throttling): 열악한 네트워크 환경 시뮬레이션<br>
사용자의 네트워크 속도를 인위적으로 제한하여 저속 인터넷 환경에서의 앱 서비스 안정성을 테스트하는 기능<br>
주요 용도:<br>
- 속도 제한: 3G, 4G, 혹은 아주 느린 카페 Wi-Fi 환경을 프리셋으로 선택하여 데이터 로딩 속도 측정.<br>
- 타임아웃 테스트: 네트워크 지연(Latency)을 길게 설정하여, 로딩 시간이 길어질 때 앱이 무한 로딩에 빠지지 않는지, 로딩 바가 정상적으로 노출되는지 확인.<br>
- 패킷 손실 시뮬레이션: 불안정한 네트워크 상황에서 데이터 일부가 유실될 때 앱이 비정상 종료(Crash)되지 않고 적절한 안내 문구를 띄우는지 테스트.<br>
<br>

# 원활한 검증을 위해 보안이 해제된 디버깅 빌드 활용이 필수적임.<br>
